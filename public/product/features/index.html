Defense in Depth

Security is not a feature. It is the foundation.

Every layer independently protects. Assume host compromised, DB exfiltrated, attacker has root. Data stays safe.

🔒

AES-256-GCM Field Encryption

PII (emails) encrypted at the field level, not just disk. Three-tier key derivation: Cloudflare Worker vault half + local .env half + HKDF derivation. Neither half alone produces the key. Stolen DB = useless ciphertext.

IV(12) + AuthTag(16) + Ciphertext · timing-safe decryption
🔐

Argon2id Password Hashing

Memory-hard hashing: 64MB memory cost, 3 iterations, 4 parallelism. No bcrypt, no scrypt. Timing-safe comparison for non-existent users (compares against pre-computed dummy hash to prevent user enumeration).

memoryCost=65536 · timeCost=3 · parallelism=4
🛡

Token Bucket Rate Limiting

O(1) per request. Runs before body parsing, before session validation, before everything. Pre-built 429 response (~180 bytes), zero serialization on reject. Separate stricter bucket for auth endpoints.

10K IP limit · LRU eviction · 200 burst / 5 auth burst
🛡

SSE Connection Guards

Per-IP cap (5) and global cap (200) on Server-Sent Event connections. Pre-built rejection response. Decrements on disconnect. Prevents connection exhaustion attacks.

Per-IP + global tracking · auto-cleanup on disconnect
🛡

Security Headers

CSP with nonce-based inline scripts. X-Frame-Options DENY. HSTS 1-year with subdomains. Permissions-Policy disables camera, microphone, geolocation. Referrer-Policy strict-origin-when-cross-origin.

No eval · no inline without nonce · frame-ancestors none
🔒

CSRF Protection

Dual validation: Origin header check against allowed origins + X-Requested-With fallback. All state-changing requests (POST, PUT, PATCH, DELETE) must pass. Both checks must fail before rejection.

Origin + XHR header · loopback-aware · logged rejections
🔐

Session Fingerprinting

SHA-256 hash of IP + User-Agent + Accept-Language + Accept-Encoding binds sessions to client identity. Mismatched fingerprint = session destroyed. Detects hijacking even with valid session ID.

256-bit random session IDs · HttpOnly + SameSite=Strict
🔒

Auth Rate Limiting

Exponential backoff on failed logins: 2^(n-3) seconds after 3 failures, capped at 1 hour. 10 failures = permanent lock (requires server restart). Rate limit key: IP + User-Agent combined.

3-fail trigger · exponential 1s→1hr · 10-fail permanent lock
🛠

Cloudflare Worker Secret Vault

10-layer defense in depth. AES-256-GCM encrypted comms, HMAC-SHA256 request signing, HKDF key splitting across vault and client. Fail-safe fallback if vault unreachable. Multi-client broker support.

10 security layers · split-key architecture · graceful degradation
📋

Append-Only Audit Trail

Every privileged operation logged: action, resource type, resource ID, actor, old value, new value, detail. Indexed by resource and timestamp. Configurable retention (default 10K rows).

Immutable log · indexed · admin-queryable
🔓

Zero innerHTML

Entire frontend, admin panel, and all dynamic rendering uses safe DOM APIs (el() helper, textContent). No innerHTML anywhere. No dynamic code execution. No raw HTML injection. XSS is structurally impossible.

el() helper · textContent only · CSP nonce enforced
🔓

Webhook URL Validation

Blocks private/internal IPs (127.*, 192.168.*, 10.*, 172.16-31.*) in webhook URLs. Prevents SSRF attacks via webhook configuration.

IP range blocklist · validated before storage

Technical Specifications

Under the hood.

Runtime

Node.js 18+ (tested 18, 22). Express. SQLite (WAL mode). Pino structured JSON logging. 4 production dependencies: argon2, better-sqlite3, express, pino.

Database

9 tables. 75+ prepared statements. WAL journaling. Soft delete. 5s busy timeout. Parameterized queries only. Zero SQL injection surface.

Architecture

Dual-server: public app (0.0.0.0:51777) + admin (127.0.0.1:random). Shared middleware. SSE for real-time. All state in SQLite. Stateless request handling.

Deployment

Docker (multi-stage Alpine, non-root). Docker Compose (resource limits, health checks). Kubernetes (Kustomize, security context, probes). Bare metal (autoconfig.sh, systemd, Nginx). CI/CD (GitHub Actions, 2×2 matrix).

Observability

Correlation IDs on every request (X-Request-Id). Pino structured JSON. Error auto-persist to DB. Deep health check (DB, disk, pipeline, errors). Heartbeat file for external monitoring.

Testing

305 tests across 5 suites: reliability (168), dependency audit (27), performance (28), data durability (35), code quality (47). CI runs on Ubuntu + Windows, Node 18 + 22.

Performance

Token bucket rate limiting: O(1). Pre-built 429 response: zero serialization. Static assets: 1-day immutable cache. Dynamic: no-store. SSE with connection caps.

License

Apache 2.0. Open source. No vendor lock-in. Full data export. Self-hosted. Your data never leaves your machine.
