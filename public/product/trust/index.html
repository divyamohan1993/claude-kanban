How We Test

Every test runs against the live server with real HTTP requests, real database operations, and real authentication flows. No mocks, no stubs, no fakes.

305 automated tests across 5 independent suites verify reliability, security, performance, data integrity, and code quality. Run node tests/run-all.js to reproduce these results yourself.

Test suites are open source. Read every test at tests/ in the repository. The JSON data behind this report is at docs/trust/.

What We Verify

March of Nines (168 tests): Startup health, API contracts, error resilience, security posture (CSRF, CSP, injection), data integrity, concurrency safety, recovery, configuration, edge cases, authentication flows.

Dependency Audit (27 tests): Vulnerability scanning, license compliance, supply chain integrity, no dynamic code execution, native binary safety, lock file consistency.

Performance (28 tests): p50/p95/p99 latency, throughput under sustained load, database stress (50 concurrent writes), SSE connection speed, static asset caching.

Data Durability (35 tests): Full CRUD lifecycle, WAL mode verification, backup system, concurrent write consistency, boundary data sizes, Unicode persistence, ID sequence integrity.

Code Quality (47 tests): File structure, structured logging (no console.log), SQL injection prevention (prepared statements), no hardcoded secrets, HTTP security headers, error handling patterns, documentation presence.
